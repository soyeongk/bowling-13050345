# Plan

## 검증할 동작
한 프레임에서 스트라이크(1회 투구로 10핀)가 나오면, 그 프레임은 1회 투구로 끝나고 점수는 10 + 다음 2회 투구 핀 수 합이어야 한다.

## 작성할 테스트
`test_game.py`에 다음 테스트를 추가한다. 1프레임 스트라이크 후 3,4를 굴리고, 나머지는 모두 거터.
프레임1 = 10+3+4 = 17, 프레임2 = 3+4 = 7, 나머지 0 → 총 24점.

```python
def test_one_strike_adds_next_two_rolls_as_bonus():
    game = Game()

    game.roll(10)
    game.roll(3)
    game.roll(4)
    for _ in range(16):
        game.roll(0)

    assert game.score() == 24
```

## 구현 계획 (테스트를 통과시키기 위한 프로덕션 코드)
`score()`의 프레임 순회 루프에서 각 프레임 시작 시 첫 투구가 10(스트라이크)인지 먼저 확인하도록 분기를 추가한다.

- `self._rolls[i] == 10`이면: 프레임은 1회 투구만 소비. `frame_score = 10 + rolls[i+1] + rolls[i+2]`. 인덱스는 `i += 1`만 진행.
- 그렇지 않으면: 기존 로직(2회 투구 소비, 합이 10이면 스페어 보너스) 그대로 유지.

## 범위 제한 (이번 사이클에서 하지 않는 것)
- 10번 프레임의 fill ball(최대 3회 투구) 규칙은 다루지 않는다.
